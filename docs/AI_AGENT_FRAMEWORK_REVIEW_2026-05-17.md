## 二、文档优化建议评估

### 2.1 高优先级优化评估

#### ✅ P5-1：启用实体别名硬规则（实施）

**评估**：完全合理。Level-1 别名硬规则代码已就绪（[`_apply_alias_redirection()`](gsuid_core/ai_core/memory/ingestion/worker.py:422)），但 [`extraction.py`](gsuid_core/ai_core/memory/prompts/extraction.py:1) 的提取提示词未产出 `alias_of` 字段，导致该功能处于休眠态。激活此功能可解决"深渊"→"深境螺旋"等词汇消歧问题。

**本次实施**：修改 [`extraction.py`](gsuid_core/ai_core/memory/prompts/extraction.py:26) 的提取提示词，新增 `a` 字段（别名指向的正式名称）和别名标注规则说明。

#### ⚠️ P5-2：增强定时任务调度（暂不实施）

**评估**：需求合理，但实现复杂度较高：
- cron 表达式支持需要改造调度系统
- "每个交易日 9:30 执行"需要市场日历概念
- 放宽循环任务最大执行次数涉及安全边界

**结论**：暂不实施，等待后续版本迭代。

#### ⚠️ P5-3：冷启动工具加载补偿（暂不实施）

**评估**：建议合理但需要更多设计。使用 LLM 进行领域分类来辅助向量检索，会增加一次 LLM 调用开销和延迟。需要更仔细的方案设计。

**结论**：暂不实施，等待进一步设计。

### 2.2 中优先级优化评估

#### ✅ P5-4：语境注入 Token 预算保护（实施）

**评估**：完全合理。当前 [`format_context_injection()`](gsuid_core/ai_core/memory/group_profile.py:167) 的词汇映射没有字符数限制，当词汇映射过多时可能挤占记忆上下文空间。

**本次实施**：在 [`group_profile.py`](gsuid_core/ai_core/memory/group_profile.py:167) 的 `format_context_injection()` 中增加 `max_chars` 参数（默认 400），在累加词汇映射条目时检查总字符数，超预算则停止添加。

### 2.3 低优先级优化评估

#### ❌ 子 Agent 结果安全过滤

**评估**：不建议实施。数据脱敏是复杂的安全问题，不应仅依赖简单规则屏蔽。SubAgent 返回的内容本身受 LLM 推理控制，在摘要阶段做字符串过滤过于简陋，可能产生误过滤或漏过滤。

**结论**：不实施。如有安全需求，应在更底层（如日志系统）做统一处理。

#### ❌ 连续无工具检测的精细控制

**评估**：当前实现已足够精细。文档指出问题，但实际情况是：
- 检测仅对"创建于 Chat/Agent"的 session 生效
- 提醒是温和的"请立即检查工具列表"
- 区分"纯闲聊"与"拒绝执行"需要额外的判断逻辑，可能引入新问题

**结论**：不实施，当前实现已平衡用户体验和功能需求。

#### ❌ 图片摘要的"置信度"标记

**评估**：不建议实施。在摘要文本后附加"（基于图片摘要）"提示，会让用户觉得信息不可靠，但实际大多数情况下摘要足够准确。这种提示更多是降低用户信任，而非真正解决问题。

**结论**：不实施。

## 三、本次实施优化

### P4-1：启用实体别名提取（alias_of）

**文件**：[`gsuid_core/ai_core/memory/prompts/extraction.py`](gsuid_core/ai_core/memory/prompts/extraction.py:26)

**修改内容**：在提取提示词的输出格式中新增 `a` 字段（别名指向的正式名称）和别名标注规则说明。

```diff
 **输出格式严格要求（使用简写键名提高处理速度）**：
 - n: 实体名称
 - s: 实体摘要(极简，不超过50字)
 - t: 标签数组（最多5个标签）
 - u: user_id (可选)
 + a: 别名指向的正式名称 (可选，若存在则表示此实体是别名，需配合 s 说明别名指向)
 - src: 关系主语
 - tgt: 关系宾语
 - f: 事实描述

 + **别名标注规则**：
 + - 当某个实体是另一个实体的别名时（如"大班尼特"是"妮可"的别名），在该实体的 a 字段填写正式名称
 + - 别名实体的 s（摘要）应说明"是XXX的别名"，如 "s": "是妮可的别名"
 + - 正式实体不需要填写 a 字段
 + - 同一正式实体可以有多个别名（如"大班尼特"、"天使"都指向"妮可"）
```

**效果**：LLM 在提取实体时会自动标注别名关

系，激活 Level-1 别名硬规则，使"深渊"→"深境螺旋"、"大班尼特"→"妮可"等映射自动生效。

### P4-2：语境注入 Token 预算保护

**文件**：[`gsuid_core/ai_core/memory/group_profile.py`](gsuid_core/ai_core/memory/group_profile.py:167)

**修改内容**：`format_context_injection()` 新增 `max_chars` 参数（默认 400），在累加词汇映射条目时检查总字符数，超预算则停止添加。

```python

    scope_key: str,
    max_chars: int = 400,
) -> str:
    """生成可注入对话的【当前群聊语境】文本。

    "深渊"在本群指什么、某个外号对应哪个角色。

    Args:
        scope_key: 群组 scope key
        max_chars: 最大字符数限制，默认 400，超出则截断词汇映射条目
    """
    # ...
    if term_mappings:
        lines.append("语境说明（群内特有词汇）:")
        # 按频次降序截断，超预算则停止添加词汇映射
        for alias, formal in list(term_mappings.items())[:12]:
            entry = f"  - \"{alias}\" = {formal}"
            if sum(len(l) for l in lines) + len(entry) > max_chars:
                break
            lines.append(entry)
    return "\n".join(lines)
```

**效果**：词汇映射条目过多时自动截断，确保语境注入文本不超过预算，保护记忆上下文空间。

## 四、潜在风险与注意事项

| 风险点 | 说明 | 严重程度 |
|--------|------|---------|
| alias_of 提取增加 LLM 负担 | 新增字段可能导致提取失败率上升 | 低（可通过后置检查兜底） |
| 语境注入截断可能丢失重要映射 | 高频映射被保留，低频映射被截断 | 低（按频次排序保证重要映射优先） |
| 定时任务循环上限仍是 10 | 长期运行任务（如虚拟炒股每日执行）可能提前终止 | 中（建议用户分拆任务或等待后续 cron 支持） |
