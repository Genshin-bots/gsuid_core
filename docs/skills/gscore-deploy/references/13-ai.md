# 十三、AI 核心部署要点

> **返回主入口**：[`../SKILL.md`](../SKILL.md) · **上一章**：[十二、Docker 部署](./12-docker.md) · **下一章**：[十四、升级与热更新](./14-upgrade.md)

GsCore 内置 **AI Core** 子系统，提供 AI 工具调用、RAG 检索、多 Agent、人格、
记忆、MCP 等能力。本章讲**部署者角度**怎么开关、配 Key、避免常见坑。

源码目录：[`gsuid_core/ai_core/`](../../../gsuid_core/ai_core/)（30+ 模块）。
**用户文档**：https://docs.sayu-bot.com/AIFeatures/（配置文档在 `Advance/AIConfig.md`）。

## 13.1 启用 AI

**默认关闭**。`data/ai_core/ai_config.json`：

```json
{
  "enable": false,
  ...
}
```

启用：

1. WebConsole → **AI 配置** → 「基础配置」 → 打开「启用AI服务」
2. 配 OpenAI 兼容模型 profile（见 §13.3）
3. 配外部服务 Key（见 §13.4）
4. `core重启`

启动日志应能看到：

```
🧠 AI工具: 12 | 🔗 Trigger工具: 5 | 🎭 人格: 3 | 📋 配置文件: 2
```

> 没看到这一行 → AI 没启用。

## 13.2 ai_config.json 字段速查

源码：[`gsuid_core/ai_core/configs/ai_config.py`](../../../gsuid_core/ai_core/configs/ai_config.py)

| 字段 | 默认 | 说明 |
|------|------|------|
| `enable` | `false` | 总开关 |
| `ai_mode` | `["提及应答"]` | AI 行动入口：`提及应答` / `定时巡检` / `趣向捕捉` / `困境救场` |
| `enable_rerank` | `false` | 启用 Rerank（提升 RAG 质量，模型约 1GB） |
| `enable_memory` | `true` | 启用记忆子系统 |
| `high_level_provider_config_name` | `""` | 高级任务（工具调用）模型 profile，格式 `provider++name`（如 `openai++MiniMAX`） |
| `low_level_provider_config_name` | `""` | 低级任务（简单问答）模型 profile |
| `embedding_provider` | `local` | 嵌入提供方（`local` / `openai`） |
| `rerank_provider` | `local` | Rerank 提供方（`local` / `openai`） |
| `qdrant_provider` | `local` | Qdrant 向量库部署（`local` 嵌入式 / `remote` 远程） |
| `websearch_provider` | `Tavily` | 网络搜索（`Tavily` / `Exa` / `MCP`） |
| `image_understand_provider` | `MCP` | 图片理解（MCP） |
| `asr_provider` | `MCP` | 语音识别（MCP） |
| `video_understand_provider` | `MCP` | 视频理解（MCP） |
| `document_extract_provider` | `MCP` | 文档提取（MCP） |
| `multi_agent_lenth` | `20` | AI 最大思考轮数（防死循环烧 token） |
| `white_list` | `[]` | AI 白名单用户 |
| `black_list` | `[]` | AI 黑名单用户 |
| `hf_endpoint` | `https://hf-mirror.com` | HuggingFace 镜像（local 嵌入下载用） |

> **`hf_endpoint` 在 Core 启动早期就被读入**（见 [`core.py:74-77`](../../../gsuid_core/core.py)），
> 改完要 `core重启`。

## 13.3 模型 Profile（OpenAI 兼容）

每个 LLM 模型是一个 **profile**，文件位置：

```
data/ai_core/
├── provider_configs/
│   ├── openai/
│   │   ├── MiniMAX.json
│   │   ├── DeepSeek.json
│   │   └── ...
│   ├── anthropic/
│   └── gemini/
├── openai_config.json       # 索引文件，列出所有 profile
├── anthropic_config.json
└── gemini_config.json
```

**profile 示例**（`MiniMAX.json`）：

```json
{
  "base_url": "https://api.MiniMAX.com/v1",
  "model_name": "MiniMax-Text-01",
  "api_key": "sk-xxx",
  "model_support": ["text"]
}
```

| 字段 | 说明 |
|------|------|
| `base_url` | API 基础 URL（OpenAI 兼容） |
| `model_name` | 模型名 |
| `api_key` | API 密钥 |
| `model_support` | `["text"]` / `["text", "image"]` / `["text", "image", "audio"]` / `["text", "image", "audio", "video"]` |

> **建议**：WebConsole → AI 配置 → 「模型配置」 直接加 / 改 profile（自动
> 校验 URL / 测连通性）。

## 13.4 外部服务 Key

### 13.4.1 Tavily（网络搜索）

- 注册：https://tavily.com
- 免费额度：1000 次 / 月
- 配置：`data/ai_core/tavily_config.json`

```json
{
  "api_key": "tvly-xxx",
  "max_results": 10,
  "search_depth": "basic"
}
```

`search_depth`：`basic`（快）/ `advanced`（更详细但慢）。

### 13.4.2 Exa（备选网络搜索）

`data/ai_core/exa_config.json`：

```json
{ "api_key": "exa-xxx", "max_results": 10 }
```

### 13.4.3 Qdrant（远程向量库）

`qdrant_provider=remote` 时配 `data/ai_core/qdrant_config.json`：

```json
{
  "url": "https://qdrant.example.com:6333",
  "api_key": "xxx"
}
```

> `local` 模式无需配置，使用嵌入式 Qdrant + LMDB 存储。

### 13.4.4 MCP Server

MCP（Model Context Protocol）提供图片理解 / 语音识别 / 视频理解 / 文档提取等
能力。`data/ai_core/mcp_config/` 下放每个 server 的配置。

详见：[`docs/MCP_SERVER.md`](../../../MCP_SERVER.md)。

## 13.5 资源消耗

| 组件 | 磁盘 | 内存 | 备注 |
|------|------|------|------|
| 嵌入模型（local） | ~300 MB | ~300 MB | 首次启动从 HF 下载 |
| Rerank 模型（local，可选） | ~1 GB | ~1 GB | 启用后吃内存 |
| Qdrant local | 数据量相关 | 与数据量相关 | 嵌入式 |
| OpenAI 兼容 LLM API | - | - | 按 token 计费 |

> **建议**：启用 `enable_rerank=true` 前确认机器 ≥ 2GB 内存。

## 13.6 部署建议

### 13.6.1 小规模（个人 / 小群）

- `embedding_provider=local` + `websearch_provider=Tavily`
- 一个 OpenAI 兼容 profile（如 DeepSeek / 智谱 GLM）
- `multi_agent_lenth=12` 省 token

### 13.6.2 中规模（生产）

- 启用 `enable_rerank=true`
- `qdrant_provider=remote`（独立 Qdrant 容器 / 服务）
- 高级任务用贵的模型（如 GPT-4 / Claude Sonnet）
- 低级任务用便宜模型（如 GPT-4o-mini / DeepSeek）

### 13.6.3 安全

- `white_list` / `black_list` 严格控制谁能用 AI（避免 token 被刷）
- `multi_agent_lenth` 不超过 30（防递归死循环）
- WebConsole 改 model profile 后**立即生效**（多数可热更）

## 13.7 故障排查

| 现象 | 原因 / 解决 |
|------|--------------|
| 启动没 AI 日志 | `enable=false` / 配置文件没读到 |
| 嵌入模型下载失败 | `hf_endpoint` 设错 / 网络限；切 `https://hf-mirror.com` |
| LLM API 调不通 | `base_url` / `api_key` 错；先 curl 测连通 |
| Rerank 启用后内存爆 | 机器太小；用 `openai` 远程 rerank |
| Web search 报 401 | Tavily Key 错或过期 |
| Qdrant remote 连不上 | URL / 端口 / api_key 错；先 `curl <url>/healthz` |
| 切换 qdrant_provider 不生效 | 启动时会自动迁移历史数据，等迁移完再测 |
| `multi_agent_lenth` 太大 token 烧光 | 调到 12 或 9 |

## 13.8 不启用 AI

AI 完全可以不开：

- `enable=false`
- 所有 `*_provider` 字段即使配了也不会被调用
- 不会下载嵌入模型（首次启动会跳过）
- Core 启动不会拉 AI 核心后台初始化

> Bot 业务功能（命令、数据库、插件）**不依赖 AI**。
